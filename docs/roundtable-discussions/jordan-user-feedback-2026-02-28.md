# Jordan Lee 使用心得回饋

**日期**: 2026-02-28
**使用期間**: 2026-02-14 ~ 2026-02-28 (2 週)
**使用情境**: Obsidian 撰寫技術文章 + Astro 部落格發布
**發表頻率**: 每週 2~3 篇文章

---

## 1. 整體滿意度

### 是否達到預期？

**7/10 分**。老實說,比我預期的好一些,但也有些地方讓我意外的卡。

**最滿意的功能**: Wiki Link 轉換和圖片處理。這兩個功能真的省了我大量時間。以前我要手動改每一個 `[[]]` 連結,還要把圖片一個一個複製到正確位置。現在點一下「同步到 Blog」就搞定了。這個價值是實實在在的。

**最不滿意的地方**: 編輯器的行為有時候有點「詭異」。不是壞掉,而是一些小細節讓我覺得不太順。比如說,我在編輯文章時,有時候會感覺自動儲存「太積極」了 —— 明明我只是在思考,游標停在某個段落,它就覺得我改了東西要存檔。雖然沒造成實際問題,但那個「正在儲存...」的提示會打斷我的思路。

另外,**文章列表的體驗還不夠好**。每次我發布完,回到列表頁,畫面會跳一下,或者我剛才選的文章不見了。我得重新找我要編輯的那篇。這個在日常使用中很煩。

---

## 2. 日常痛點

### 發布流程是否順暢？

**8/10 分**。流程本身很順,實際上就是:

1. 在 Obsidian 寫完
2. 打開 WriteFlow
3. 把文章狀態改成 `published`
4. 點「同步到 Blog」
5. 完成

這個流程本身沒問題,**但我希望能少一個步驟** —— 如果能在編輯器裡直接改狀態 (像是有個按鈕「發布」),就不用切回文章列表了。現在的流程是:

```
編輯器 → 列表頁 → 改狀態 → 同步 → (如果還要改東西) → 列表頁 → 點文章 → 編輯器
```

有點繞。如果是:

```
編輯器 → 點「發布」→ 同步 → 繼續編輯
```

會更直覺。

### 編輯器的寫作體驗如何?

**7.5/10 分**。速度沒問題,Markdown 語法高亮也很好。但有兩個地方會打斷我的思路:

1. **自動儲存提示太頻繁**
   我知道團隊想讓我安心 (顯示「已儲存」),但這個提示出現太頻繁了。尤其是我在思考時,游標停在某段文字上,它會突然跳出「正在儲存...」。我知道這可能是因為我無意間按了什麼鍵或鼠標,但作為使用者我不想看到這麼多「虛驚」。

   **建議**: 把「正在儲存」的提示改成更不顯眼的方式 (比如只在狀態欄顯示一個小圖示),或者延遲 1-2 秒才顯示,避免「閃現」。

2. **Wiki Link 自動完成有時候會卡**
   當我輸入 `[[` 時,下拉選單會出現,但有時候反應有點慢 (大概 1-2 秒)。我的 Vault 裡有大約 200 篇文章,不算多,但可能是搜尋邏輯有點重?

   這不是大問題,因為我可以直接打完整名稱,但如果能優化會更好。

### 自動儲存是否可靠？

**9/10 分**。沒遇到未儲存的情況,這點很好。但就像上面說的,**誤觸發的頻率有點高**。

有幾次我只是:
- 點了一下編輯器 (切換焦點)
- 用滑鼠選取一段文字 (準備複製)
- 游標停在某個位置思考

結果它就顯示「正在儲存...」。雖然最終沒有真的存 (因為沒改動),但那個提示還是會跳出來。

**這不影響功能,但影響體驗**。我希望能更「安靜」一點。

## 3. 緊急功能需求 (Top 3)

### Priority 1: 編輯器內直接發布

**為什麼重要**:
每次我寫完一篇文章,我的流程是:
1. 寫完
2. 預覽一遍
3. 改 Frontmatter (加 tags, 確認 category)
4. 發布

現在我要「退出編輯器 → 找到文章 → 改狀態 → 同步」,太繞了。

**希望的功能**:
- 在編輯器右上角有個「發布」按鈕 (或鍵盤快捷鍵)
- 點下去就把這篇文章改成 `published` 並自動觸發同步
- 不用離開編輯器

**沒有這功能我能接受嗎？**
可以,但很煩。如果下個 Sprint 能做,我會很開心。

---

### Priority 2: 文章列表的穩定性

**為什麼重要**:
這是我每天最頻繁操作的地方:
- 找文章
- 切換文章
- 改狀態

但現在的列表有幾個問題:

1. **發布後畫面會跳**
   同步完成後,列表會重新載入,有時候我剛才選的文章不見了,得重新找。

2. **排序不一致**
   有時候按「最後修改時間」排,有時候又變成「建立時間」,我搞不清楚現在是哪個邏輯。

3. **找文章很慢**
   我有 200 篇文章,但列表沒有「釘選」或「最近編輯」這種快速篩選。我每次都要滾動或用搜尋。

**希望的功能**:
- 同步完成後,**保持我當前選取的文章**
- 明確的排序邏輯 (最好是「最近編輯」優先,或讓我自己選)
- 「最近編輯」快速篩選區 (顯示最近 5 篇)

**沒有這功能我能接受嗎？**
勉強可以,但這真的影響日常效率。如果能改,我會超級感激。

---

### Priority 3: 發布後自動 Git Push

**為什麼重要**:
現在的流程是:
1. WriteFlow 同步到本地 Astro 資料夾
2. 我手動開 Terminal
3. `git add .`
4. `git commit -m "發布文章"`
5. `git push`
6. 等 Netlify 自動部署

**這個流程完全可以自動化**。我看到 MVP 文件說這是 v0.2 的功能,但老實說,**這個對我來說比很多「表面功能」更重要**。

**希望的功能**:
- 同步完成後,自動執行 Git 操作 (背景執行即可)
- 給我一個選項「發布後自動 push」(預設開啟)
- 如果失敗了,給我一個通知

**沒有這功能我能接受嗎？**
可以,因為我現在已經習慣手動 push 了。但這是遲早要做的功能,早點做我會更開心。

## 4. 可以接受的不足

### 「其實沒那麼重要」的功能

1. **專注模式 (Focus Mode)**
   我以為我會很常用,但實際上我沒開過幾次。對我來說,左邊的文章列表留著也沒差,我不是那種需要「全視窗寫作」的人。

   **建議**: 後續版本再優化,不急。

2. **主題切換 (Dark Mode)**
   我以為我會在意,但其實我大部分時間都用預設主題。只要不是太醜就好。

   **建議**: 有個深色模式當然好,但不是 Priority。

3. **大量客製化選項**
   比如說「自訂字體」、「自訂行高」、「自訂顏色」這種。我知道有些使用者會喜歡,但對我來說,**只要預設的好用就夠了**。

   **建議**: 不要花太多時間在這種「Nice to have」的功能上。

### 可以忍受的 Bug 或缺陷

1. **自動儲存的虛驚**
   雖然上面抱怨了,但這不會讓我停用這個 App。我可以忍受,只是希望下個版本能改善。

2. **Wiki Link 自動完成的延遲**
   1-2 秒的延遲可以接受。如果要花很多時間優化,可以放後面。

3. **UI 不夠美**
   說實話,UI 算不上漂亮,但夠用。我不是來看介面的,我是來發布文章的。功能穩定比好看重要。

## 6. 使用建議

### 新使用者可能踩的坑

1. **第一次設定路徑時,不知道要填什麼**
   我第一次打開設定介面時,看到「Obsidian Vault 路徑」和「Astro Blog 路徑」,我不確定:
   - 要填「資料夾路徑」還是「檔案路徑」？
   - Astro 路徑是填「專案根目錄」還是「content 資料夾」？

   **建議**: 在輸入框下方加個小提示,比如:
   ```
   例如: C:\Users\Jordan\Documents\ObsidianVault
   提示: 請選擇 Vault 的根目錄資料夾
   ```

2. **不知道「同步到 Blog」會做什麼**
   我第一次按「同步到 Blog」時,不確定它會:
   - 只同步這一篇文章？還是所有 `published` 文章？
   - 會不會覆蓋我之前手動改過的內容？

   **建議**: 在按鈕旁邊加個說明,或者第一次使用時跳出一個「提示視窗」:
   ```
   「同步到 Blog」會將所有標記為 published 的文章同步到 Astro 資料夾。
   已存在的文章會被覆蓋。是否繼續？
   ```

3. **文章狀態 (draft / published) 的邏輯不清楚**
   我一開始以為「draft」的文章就不會出現在列表裡,但實際上是「都會出現,只是不會同步」。

   **建議**: 在文章列表上方加個篩選器 (顯示全部 / 只顯示 published / 只顯示 draft)。

### UI/UX 改善空間

1. **自動儲存的回饋太顯眼**
   建議改成:
   - 只在狀態欄顯示小圖示 (💾 → ✅)
   - 或者延遲 1-2 秒才顯示,避免「閃現」

2. **文章列表的視覺層級不清楚**
   現在看起來所有文章都長一樣,我很難快速找到「我最近在編輯的那篇」。

   **建議**:
   - 高亮「最近編輯」(比如顯示「2 分鐘前」)
   - 或者讓「當前編輯中」的文章有個明顯的標記

3. **錯誤訊息不夠友善**
   有一次我設定了錯誤的路徑,同步失敗了,錯誤訊息是:
   ```
   Error: ENOENT: no such file or directory
   ```
   作為使用者,我看到這個訊息會覺得:「所以...我該怎麼辦？」

   **建議**: 把錯誤訊息改成:
   ```
   無法找到 Astro Blog 路徑。
   請檢查設定中的「Astro Blog 路徑」是否正確。
   (設定 → 一般 → Astro Blog 路徑)
   ```

如果下個 Sprint 能解決我的 P1 和 P2,我會覺得 WriteFlow 從「可用」升級到「好用」。
