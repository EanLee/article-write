# 決策記錄：部落格改善與收入轉化策略

> **話題編號**: topic-012
> **決策日期**: 2026-02-27
> **決策類型**: 多數共識
> **相關討論**: [discussion.md](./discussion.md)

---

## 決策結果

**最終決定**:
以「工程師效率工作流」為核心主題定位，分三階段推進部落格改善與商業化：第一階段（0-3 個月）建立基礎設施（SEO 監控、電子報、Affiliate）；第二階段（3-9 個月）強化內容策略（主題群、系列文章、旗艦內容）；第三階段（9 個月+）推出付費產品（電子書、課程或 WriteFlow 付費版）。

**決策理由**:
1. 四位外部專家一致認為部落格有足夠潛力，但缺乏系統性的 SEO 基礎和商業化路徑
2. WriteFlow 工具的開發本身就是最有說服力的「Build in Public」素材
3. 電子報是唯一「不依賴算法」的受眾資產，是商業化的必要前提
4. Affiliate 聯盟行銷是零成本、立即可行的第一步被動收入

## 核心決策：部落格定位

**選定定位**: 「工程師效率工作流」

| 面向 | 內容 |
|------|------|
| **目標受眾** | 工程師、技術部落客、軟體從業人員 |
| **核心主題** | Obsidian + Astro 工作流、知識管理、寫作效率 |
| **輔助主題** | 工程師成長思考、技術管理心得 |
| **差異化** | 作者自己開發工具（WriteFlow），提供第一手工作流實戰 |
| **品牌聲音** | 務實、深度、有個人觀點，非教程搬運站 |

## 三階段商業化路徑

### 階段一：基礎建設（0-3 個月）

| 項目 | 預估效益 | 投入時間 |
|------|---------|---------|
| Google Search Console 監控 | 了解 SEO 現況，找出優化機會 | 30 分鐘 |
| 電子報訂閱系統 | 建立「有主權」的讀者資產 | 2-4 小時 |
| Affiliate 聯盟連結 | 每月 NT$500-5,000 被動收入 | 2-3 小時/次 |
| 首頁讀者引導優化 | 降低跳出率，增加頁面停留 | 4-8 小時 |
| RSS Feed 確認 | 讓閱讀器用戶能訂閱 | 1 小時 |

### 階段二：內容強化（3-9 個月）

| 項目 | 預估效益 | 投入時間 |
|------|---------|---------|
| 發布日曆制定 | 一致性發布，讀者期待感 | 1-2 小時規劃 |
| 「工程師效率工作流」主題群 | SEO 流量積累，建立話題權威 | 持續寫作 |
| 5 篇系列文章（完整規劃後發布）| 讀者黏著度提升，訂閱率提升 | 20-30 小時/系列 |
| WriteFlow 開發日誌（Build in Public）| 工具曝光，社群互動 | 每週 1-2 小時 |

### 階段三：付費產品（9-18 個月）

| 項目 | 預估月收入 | 備註 |
|------|----------|------|
| 付費電子報（訂閱制）| NT$3,000-15,000 | 需要先有 500+ 免費訂閱者 |
| 電子書（Gumroad）| NT$5,000-20,000 | 「工程師知識管理完全指南」 |
| WriteFlow 付費版 | NT$10,000-50,000 | 取決於工具完成度和定位 |
| 線上課程 | NT$30,000-100,000 | 需要先有足夠的品牌知名度 |

## 追蹤

**驗證方式**:
- 每月檢視 Google Search Console 的流量變化
- 每季統計電子報訂閱人數成長
- 每季統計 Affiliate 收入
- 6 個月後評估是否進入階段二產品開發

**回顧日期**: 2026-05-27（3 個月後）

**成功指標**（3 個月目標）:
- Google Search Console 安裝並收集到足夠數據
- 電子報訂閱人數達到 50+
- Affiliate 開始產生任何收入（哪怕 NT$100）
- 發布日曆已執行 6 週以上
