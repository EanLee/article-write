## 四個核心瓶頸

| # | 瓶頸 | 根因 | 優先 |
|---|------|------|------|
| 1 | **無可交付物**（無安裝包）| 打包從未被列為 Sprint P0 | P0 |
| 2 | **進度不透明**（Done 定義鬆散）| 狀態散落 T-NNN / decision.md，無統一視圖 | P0 |
| 3 | **錯誤路徑缺失**（happy path 文化）| sad path 不在 Done 定義裡 | P0 |
| 4 | **文件與實作脫節**（文件未入流程）| 技術評估文件存在但無 checklist 強制引用 | P1 |
