## 決策結果

**最終決定**：將四個 UX 問題依技術難度分兩波處理。排版、圖片預覽為 P0，本 Sprint 修復（工作量小、效果立竿見影）；編輯器 UX 需先做技術選型評估，列入 v0.2 規劃。

---

## 四個問題確認

| # | 問題 | 根本原因 | 難度 | 優先級 |
|---|------|---------|------|--------|
| UX-01 | Preview 整體排版如純文字（標題、段落、清單無樣式）| `@tailwindcss/typography` 未安裝，`prose` class 無效 | 小 | P0 |
| UX-02 | 編輯器輸入 UX 遠不如 Obsidian（無補全、無縮排、無語法提示）| `EditorPane` 為原生 `<textarea>`，缺少編輯器功能層 | 大 | P1（v0.2）|
| UX-03 | 預覽區看不到圖片（`![[image.png]]` 空白）| 預覽 pipeline 未套用 `ObsidianSyntaxService` 的圖片轉換 | 小 | P0 |
| UX-04 | 部分 Markdown 語法未正確渲染（如 callout、特定 Obsidian 語法）| 缺少對應 markdown-it plugin 或轉換邏輯 | 中 | P0（隨 UX-01 修復時一併排查）|
