## 📋 執行摘要

### 使用者背景

我是一名使用 Obsidian 寫技術筆記的軟體工程師，想要將筆記發布到個人技術部落格（使用 Astro）。

### 核心發現

✅ **做得好的地方**
- 編輯器直觀易用，類似 Obsidian
- 文章管理清楚，搜尋和篩選功能實用
- 設定面板指引明確，3 步即可完成基本設定

⚠️ **可以改進的地方**
- 首次使用沒有引導，需要自己摸索
- 沒有範例文章或教學
- 某些功能藏得太深（例如 Undo/Redo 沒有按鈕）

❌ **重大問題**
- **核心功能（發布）尚未實作** - 無法實際使用
- 錯誤訊息不夠清楚
- 沒有使用文件或說明

## 🎯 使用場景評估

### 場景 1: 首次使用

#### 期望流程
```
1. 下載並安裝應用程式
2. 看到歡迎畫面，了解這是做什麼的
3. 跟著引導完成設定（選擇資料夾）
4. 看到範例文章，試著編輯
5. 點擊「發布」，看到文章成功發布
6. ✅ 理解如何使用，開始正式創作
```

#### 實際體驗
```
1. 下載並安裝應用程式 ✅
2. 開啟後看到空白的編輯器 ❓
   → 不知道要做什麼
   → 沒有任何提示或引導
3. 點擊齒輪圖示開啟設定 🤔
   → 看到「部落格設定」，猜測要設定路徑
4. 設定文章資料夾和部落格路徑 ✅
   → 有必填標示，清楚
   → 但不確定設定是否正確
5. 儲存設定後... 然後呢？ ❓
   → 沒有下一步指引
6. 試著在編輯器打字 ✅
   → 編輯器反應流暢
7. 尋找「發布」按鈕 ❌
   → 找不到（因為還沒實作）
```

**評分**: 4/10
**問題**: 缺乏引導，首次使用體驗不佳

### 場景 2: 日常寫作

#### 期望流程
```
1. 開啟應用程式
2. 點擊「新增文章」或選擇現有文章

5. 完成後點擊「發布」
6. ✅ 文章出現在部落格上
```

#### 實際體驗（編輯器部分）
```
1. 開啟應用程式 ✅
2. 左側文章列表看到所有文章 ✅
   → 可以搜尋、篩選、排序

   → 編輯器反應快速
   → Markdown 語法高亮清楚
4. 輸入 Wiki Link [[文章名稱]] ✅
   → 即時預覽顯示正確
5. 自動儲存運作正常 ✅
   → 狀態列顯示「已儲存」
6. Ctrl+Z 撤銷... ✅

**優點**: 編輯體驗流暢，功能完整

```

## 🐛 實際使用遇到的問題

### 高嚴重度問題

#### 1. 文章列表會跳動 ⛔

**重現步驟**：
1. 編輯文章
2. 儲存（Ctrl+S）
3. 文章列表會重新排序，導致選中的文章改變

**影響**：
- 嚴重影響使用體驗
- 造成困擾和不信任感

**期望**：
- 儲存後列表不應該跳動
- 如果要重新排序，應該保持當前選中的文章

### 中嚴重度問題

#### 2. 沒有使用文件 ⚠️

**問題**：
- 不知道有哪些功能
- 不知道快捷鍵
- 遇到問題不知道怎麼解決

**期望**：
- 應用程式內建幫助文件
- 或者提供線上文件連結
- 至少要有 README

#### 3. 錯誤訊息不清楚 ⚠️

**範例**：
設定錯誤的路徑時，只顯示「路徑無效」

**期望**：
- 更明確的錯誤訊息：「路徑不存在：C:\...\articles」
- 建議的解決方案：「請確認路徑是否正確，或點擊『選擇資料夾』重新選擇」

### 低嚴重度問題

#### 4. 沒有快捷鍵列表

**問題**：
- 不知道有哪些快捷鍵
- 要透過試誤學習

**期望**：
- 設定面板有「快捷鍵」Tab
- 或者按 `?` 顯示快捷鍵列表
